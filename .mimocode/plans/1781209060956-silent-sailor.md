# Plan: Replace Miro with Custom Chemical Whiteboard

## Context
The project is a chemistry tutoring bot (–•–∏–º–¢—å—é—Ç–æ—Ä). Currently uses Miro API for lesson boards, but Miro can't render chemical formulas natively and requires paid accounts. User wants the bot to write chemical formulas, equations, and reaction mechanisms on a board. Decision: replace Miro with a custom HTML whiteboard served via WebSocket, supporting three content types: structural formulas (SMILES‚ÜíRDKit.js), reaction equations (mhchem/KaTeX), and mechanism diagrams (SVG from Ketcher).

## Architecture Overview

```
Bot (Python)  ‚îÄ‚îÄWebSocket‚îÄ‚îÄ>  Whiteboard Server (Node.js)  ‚îÄ‚îÄWebSocket‚îÄ‚îÄ>  Student Browser
                                     ‚îÇ
                                     ‚îú‚îÄ‚îÄ RDKit.js (SMILES ‚Üí 2D structure)
                                     ‚îú‚îÄ‚îÄ KaTeX + mhchem (equation rendering)
                                     ‚îî‚îÄ‚îÄ SVG assets (mechanism diagrams)
```

**Key change**: Replace `BaseMiroClient.execute(action_str)` with `BoardClient.execute(board_command_json)` that sends WebSocket messages to the whiteboard server.

## Phase 1: Whiteboard Backend (WebSocket Server)

### New file: `bot/src/board/server.py`
- AsyncIO WebSocket server using `websockets` library
- Rooms: one per lesson session (`session_id`)
- Protocol: JSON messages over WebSocket

**Board commands** (bot ‚Üí student):

## Phase 2: Whiteboard Frontend

### New directory: `whiteboard/` (new service)
- `whiteboard/package.json` ‚Äî dependencies: `rdkit.js` (via CDN), `katex`, `ws`
- `whiteboard/server.js` ‚Äî Express + WebSocket server
- `whiteboard/public/index.html` ‚Äî the whiteboard page
- `whiteboard/public/js/board.js` ‚Äî WebSocket client + rendering logic
- `whiteboard/public/js/formulas.js` ‚Äî RDKit.js SMILES rendering
- `whiteboard/public/js/equations.js` ‚Äî KaTeX + mhchem rendering
- `whiteboard/public/js/mechanisms.js` ‚Äî SVG loading + CSS animation

**Rendering pipeline in browser:**
1. `show_formula` ‚Üí RDKit.js `get_mol(smiles)` ‚Üí SVG ‚Üí draw on canvas
2. `show_equation` ‚Üí mhchem `\ce{equation}` ‚Üí KaTeX render ‚Üí insert as HTML
3. `show_svg` ‚Üí fetch `/assets/{asset_id}` ‚Üí insert SVG ‚Üí trigger CSS animations

### New file: `whiteboard/Dockerfile`
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package.json .
RUN npm ci
COPY . .
EXPOSE 3001
CMD ["node", "server.js"]
```

### Changes to `docker-compose.yml`

