# Plan: Replace Miro with Custom Chemical Whiteboard

## Context
ХимТьютор — бот-репетитор по химии. Заменяем Miro на собственную HTML-доску с WebSocket, чтобы бот мог писать химические формулы, уравнения реакций и механизмы. LLM (Claude) генерирует контент доски динамически на основе RAG из PDF-конспектов в базе знаний.

## Architecture

```
PDF конспекты → Knowledge Base (S3 + pgvector)
                     ↓ RAG
Bot (Claude LLM) → generates board commands → WebSocket → Whiteboard (student browser)
                                                      ↑
Student browser ← renders: RDKit.js (formulas) | KaTeX+mhchem (equations) | SVG (mechanisms)
```

**Workflow:**
1. Преподаватель загружает PDF-конспекты в базу знаний через веб-кабинет
2. При создании урока назначается тема → бот находит материалы через RAG
3. LLM читает материалы и генерирует: текст для TTS + команды для доски
4. Команды летят по WebSocket на белую доску → ученик видит формулы/уравнения/механизмы

## Content Types

| Тип | Источник | Рендер в браузере |
|-----|----------|-------------------|
| Структурные формулы | SMILES (из LLM) | RDKit.js → SVG |
| Уравнения реакций | mhchem нотация | KaTeX + mhchem → HTML |
| Механизмы реакций | SVG-файлы (Ketcher) | Inline SVG + CSS-анимация |

## Implementation Phases

### Phase 1: Whiteboard Backend (WebSocket Server)

**New: `whiteboard/` directory** (отдельный Docker-сервис)

- `whiteboard/package.json` — Express + ws
- `whiteboard/server.js` — HTTP (static files) + WebSocket server
- `whiteboard/public/index.html` — белая доска
- `whiteboard/public/js/board.js` — WebSocket client + canvas/SVG rendering
- `whiteboard/public/js/formulas.js` — RDKit.js SMILES rendering (CDN)
- `whiteboard/public/js/equations.js` — KaTeX + mhchem rendering
- `whiteboard/public/js/mechanisms.js` — SVG loading + animation
- `whiteboard/public/css/board.css`
- `whiteboard/Dockerfile`

**WebSocket protocol** (bot → student):
```json
{"type": "show_formula", "smiles": "CC(=O)O", "label": "Уксусная кислота", "x": 200, "y": 150}
{"type": "show_equation", "equation": "CH4 + 2O2 -> CO2 + 2H2O", "label": "Горение метана"}
{"type": "show_svg", "asset_url": "/assets/mechanism_sn2.svg", "animate": true}
{"type": "draw_text", "text": "Водородная связь", "x": 100, "y": 100, "style": {"fontSize": 24}}
{"type": "highlight", "target": "formula_1"}
{"type": "clear"}
{"type": "clear_step"}
```

**Rooms:** `ws://whiteboard:3001/rooms/{session_id}` — изоляция по сессии.

### Phase 2: Bot — Board Client

**New: `bot/src/board/` directory**

- `bot/src/board/__init__.py`
- `bot/src/board/client.py`:
  - `BaseBoardClient` (ABC) — `async execute(command: dict)`, `async close()`
  - `BoardClient` — WebSocket client, sends JSON commands to whiteboard server
  - `StubBoardClient` — logs commands (dev mode)
  - Factory: `make_board_client(session_id)`

**Modify: `bot/src/orchestrator/runner.py`**
- `_init_miro()` → `_init_board()` — creates BoardClient with session_id
- `self._miro` → `self._board`
- Main change: LLM now generates board commands as part of its response

**New approach to board content generation:**
Runner's `_conduct_lesson()` changes from executing pre-defined `miro_action` to:
1. LLM receives topic + RAG context + conversation history
2. LLM response includes both speech text AND board commands
3. Bot speaks the text via TTS AND sends board commands via WebSocket

**Modified lesson step format** (from LLM, not pre-defined JSON):
```python
# LLM response parsed into:
{
    "speech": "Алканы имеют общую формулу CₙH₂ₙ₊₂. Рассмотрим метан.",
    "board_commands": [
        {"type": "show_formula", "smiles": "C", "label": "Метан CH₄", "x": 300, "y": 200},
        {"type": "show_equation", "equation": "CH4 + 2O2 -> CO2 + 2H2O"}
    ],
    "question": "Какую общую формулу имеют алканы?"
}
```

**Changes to `bot/requirements.txt`:**
- Add `websockets>=12.0`

**Changes to `bot/config/settings.py`:**
- Remove `miro_access_token`
- Add `board_ws_url: str = os.getenv("BOARD_WS_URL", "ws://whiteboard:3001")`
