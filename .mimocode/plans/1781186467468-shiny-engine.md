## ДОПОЛНИТЕЛЬНО: Auto-reconnect VCS

### Задача: Бот должен переподключаться к конференции при потере соединения

**Текущее поведение:** Если соединение с VCS разрывается (закрытие вкладки, кик из конференции, сетевой сбой), бот падает с ошибкой и урок помечается как FAILED.

**Требуемое поведение:** При потере соединения бот пытается переподключиться N раз с exponential backoff. Если все попытки исчерпаны — помечает урок как FAILED.

**Где менять:**
- `bot/src/vcs/base.py` — добавить reconnect в disconnect/error handler
- `bot/src/orchestrator/runner.py` — обернуть `_connect_vcs()` и `_conduct_lesson()` в retry loop
- `bot/src/vcs/client.py` — возможно добавить метод `is_alive()` для проверки соединения

**Логика:**
```
1. Ошибка соединения → caught in runner
2. Попытка reconnect (max 3, backoff: 5s, 15s, 45s)
3. Если reconnect успешен → продолжить урок с текущего шага
4. Если все попытки исчерпаны → FAILED + event_bus publish
```

**Файлы для изменения:**
- `bot/src/orchestrator/runner.py` — основная логика retry
- `bot/src/vcs/base.py` — метод `reconnect()`
